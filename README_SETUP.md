# Agentic RAG for Smart Tourism — Chiang Mai
### Installation and Project Setup Guide (Follow Steps in Order)

---

## Quick Setup (Complete These Steps First)

This project runs entirely on a free stack with no associated costs. Only two API keys are required, both of which are available completely free of charge.

### 1. Obtain the Two Required API Keys

#### YouTube Data API v3 (Free)
1. Go to https://console.cloud.google.com/
2. Create a new project by clicking "New Project" and naming it "chiangmai-rag".
3. Navigate to **APIs & Services > Library**.
4. Search for "YouTube Data API v3" and click **Enable**.
5. Go to **APIs & Services > Credentials**.
6. Click **Create Credentials > API Key**.
7. Copy the generated key, which begins with AIza...

> **Note:** If a YouTube API Key is unavailable, Step 1 will automatically fall back to the included curated dataset (offline mode).

#### Groq API Key (Free — No Credit Card Required)
1. Go to https://console.groq.com/
2. Click **Sign Up** and register using Google or GitHub.
3. Navigate to **API Keys** from the left-hand menu.
4. Click **Create API Key** and name it "chiangmai-rag".
5. Copy the generated key, which begins with gsk_...

> Groq provides a free quota of 14,400 requests per day, which is more than sufficient for this project.

---

### 2. Install Python and Dependencies

First, verify that Python 3.10 or higher is installed:
```bash
python3 --version
```

If Python is not yet installed, download it from https://www.python.org/downloads/

To install the required packages, run the following commands:
```bash
cd agentic_rag_chiangmai
pip3 install -r requirements.txt
```

> **Note:** The installation process may take 5–10 minutes to complete. This includes `sentence-transformers`, which will download a model of approximately 275 MB the first time Step 3 is executed.

---

### 3. Configure API Keys

Copy the `.env.example` file to create a new `.env` file:
```bash
cp .env.example .env
```

Open the `.env` file using a text editor such as TextEdit or VS Code, then enter your keys as follows:
```
YOUTUBE_API_KEY=AIza...your key here...
GROQ_API_KEY=gsk_...your key here...
```

> If no YouTube API Key is available, leave the field blank or remove that line entirely. Step 1 will automatically use the curated dataset as a fallback.

---

### 4. Run the Pipeline Steps in Order

**Step 1 — YouTube Data Collection and Sentiment Analysis:**
```bash
python3 step1_youtube.py
```
> Estimated runtime: 5–10 minutes (approximately 30 seconds in offline mode).
> Output: `data/sentiment_data.csv`
>
> Model used: `cardiffnlp/twitter-xlm-roberta-base-sentiment` (~1 GB, downloaded on first run).
> Filters applied: (1) English-language comments only, (2) place-relevant content only, and (3) confidence score of 0.70 or above.

**Step 2 — GPS Hotspot Generation:**
```bash
python3 step2_gps.py
```
> Estimated runtime: 2–3 minutes.
> Output: `data/hotspots.geojson`, `data/gps_points.csv`
>
> Algorithm: Correlated Random Walk (CRW) combined with DBSCAN clustering.
> Data source: OSM Trackpoints API, with fallback to CRW simulation.

**Step 3 — Knowledge Base and FAISS Index Construction:**
```bash
python3 step3_rag.py
```
> Estimated runtime: 3–5 minutes (including a first-time model download of approximately 275 MB).
> Output: All files within the `knowledge_base/` directory.
>
> Embedding model: `paraphrase-multilingual-MiniLM-L12-v2` (Sentence Transformer, 384 dimensions).
> Fallback: TF-IDF character n-grams with TruncatedSVD (256 dimensions), used when `sentence-transformers` is not installed.
> Index type: FAISS IndexFlatIP (inner product equivalent to cosine similarity after L2 normalisation).

**Step 4 — Launch the Web Application:**
```bash
streamlit run app.py
```
> Open a browser and navigate to http://localhost:8501

## Running with Docker (Optional)

For users who prefer to run the application in a container without installing Python locally, follow the steps below.

```bash
# Step 1: Create the .env file first (refer to Step 3 above)

# Step 2: Build and run
docker compose up --build

# Step 3: Open a browser and navigate to http://localhost:8501
```

> **Note:** The pipeline steps (step1 through step3) must be executed on the host machine first in order to generate the required files in `data/` and `knowledge_base/`. The Docker container runs `app.py` (the Streamlit UI) only. The HuggingFace model cache is persisted via a named volume called `huggingface_cache`, which means models do not need to be re-downloaded on subsequent container restarts. A `start_period` of 120 seconds has been configured to allow sufficient time for the model to load on its first startup.

## Troubleshooting

| Issue | Resolution |
|-------|-----------|
| `ModuleNotFoundError` | Re-run `pip3 install -r requirements.txt` |
| `ImportError: embeddings.py not found` | Verify that `embeddings.py` is located in the same directory as `step3_rag.py` and `app.py` |
| `YOUTUBE_API_KEY not found` | This is expected — step1 will automatically use the curated dataset. Alternatively, check the `.env` file |
| `groq.AuthenticationError` | Verify that `GROQ_API_KEY` in the `.env` file begins with `gsk_` |
| YouTube quota exceeded | Wait 24 hours, or proceed directly to step2 and step3 using the curated dataset |
| `knowledge_base/faiss.index not found` | Run `python3 step3_rag.py` before launching the app |
| `data/hotspots.geojson not found` | Run `python3 step2_gps.py` before step3 |
| Model download is slow (~275 MB) | This is expected on the first run — the model is cached at `~/.cache/huggingface/` for all subsequent runs |
| Application is slow on first query | This is expected due to embedding model warm-up on initial load |
| `pickle.UnpicklingError` | Delete `knowledge_base/vectorizer.pkl` and re-run `step3_rag.py` |
